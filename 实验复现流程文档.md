# MoAL (Momentum-based Analytical Learning) 实验复现流程文档

## 📚 论文信息

**标题**: Knowledge Memorization and Rumination for Pre-trained Model-based Class-Incremental Learning

**会议**: CVPR 2025

**核心方法**: MoAL - Momentum-based Analytical Learning

**研究目标**: 在类增量学习（Class-Incremental Learning, CIL）场景下，结合预训练模型（Pre-Trained Models, PTMs），通过知识记忆和知识反刍机制，实现对新类的良好适应性（plasticity）和对旧类知识的稳定保持（stability）。

---

## 🎯 核心创新点

1. **分析学习头（Analytical Classification Head）**: 使用最小二乘解实现递归知识获取
2. **动量权重插值（Momentum-based Adapter Weight Interpolation）**: 通过EMA机制实现对新类的自适应，同时遗忘过时知识
3. **知识反刍机制（Knowledge Rumination）**: 利用精炼的适应性重新访问和强化旧知识
4. **双分支网络架构**: 结合Cosine分类器和Analytical Learning分类器

### 核心超参数说明

| 参数 | 作用 | 推荐值 | 消融实验影响 |
|------|------|--------|-------------|
| **alpha** | EMA动量系数，控制权重插值速度 | 0.999 | 越大越保守，越小适应越快 |
| **lambda_fkd** | 知识蒸馏损失权重 | 0-1 | 控制对旧知识的保持 |
| **cali_weight** | 知识反刍权重 | 0-1.0 | 控制知识反刍机制强度 |
| **rg** | 参数正则化系数 | 0-0.1 | 防止适配器参数过度偏移 |
| **Hidden** | AL隐藏层维度 | 15000-20000 | 影响表征能力 |

## 🔬 训练过程详解

### 整体流程图

```
开始
  ↓
任务 0 (初始任务)
  ├─ 加载预训练 ViT + Adapter
  ├─ 微调适配器 (tuned_epoch=80)
  ├─ 构建双分支网络 (Cosine + AC)
  ├─ 初始化 Analytical Learning 分类头
  ├─ 计算类原型均值
  └─ cls_align: 使用最小二乘求解初始权重
  ↓
任务 1-N (增量任务)
  ├─ 扩展分类头到新类别数
  ├─ 冻结 AC 模型
  ├─ 训练适配器 (progreesive_epoch=80)
  │   ├─ Cross-Entropy Loss
  │   ├─ Knowledge Distillation Loss (如果 lambda_fkd > 0)
  │   ├─ Parameter Regularization (如果 rg > 0)
  │   └─ EMA 更新 (如果 alpha > 0)
  ├─ 计算新类原型均值
  ├─ IL_align: 增量更新 AL 权重
  ├─ (可选) 知识反刍机制:
  │   ├─ cali_prototye_model: 校准原型模型
  │   ├─ _compute_relations: 计算新旧类关系
  │   ├─ _build_feature_set: 构建反刍特征集
  │   └─ cali_weight: 基于错误预测校准权重
  └─ 评估当前任务性能
  ↓
结束：输出所有任务的平均准确率
```

### 第一个任务 (Task 0) 详细流程

**目标**: 在初始类别上微调预训练模型，建立基础知识

1. **网络初始化**
   ```python
   # 加载预训练 ViT-B/16 + Adapter
   self._network = SimpleVitNet_AL(args, True)
   # 初始化 Cosine 分类器
   self._network.fc = CosineLinear(feature_dim=768, num_classes=init_cls)
   ```

2. **适配器微调** (`_init_train`)
   - 训练轮数: `tuned_epoch` (默认80)
   - 损失函数: Cross-Entropy
   - 优化器: SGD (momentum=0.9)
   - 学习率调度: CosineAnnealing
   - 只训练 Adapter 参数，冻结 ViT backbone

3. **构建双分支网络** (`construct_dual_branch_network`)
   ```python
   # 创建两个分支:
   # Branch 1: Cosine Classifier (用于训练)
   # Branch 2: Analytical Classifier (用于推理)
   network = MultiBranchCosineIncrementalNet_adapt_AC(args, True)
   ```

4. **初始化 Analytical Learning 分类头** (`cls_align`)
   - 提取训练集特征
   - 计算自相关矩阵: `auto_cor = Φ^T Φ`
   - 计算交叉相关矩阵: `crs_cor = Φ^T Y`
   - Ridge regression 优化: 选择最佳正则化参数 λ
   - 求解最小二乘: `W = (Φ^T Φ + λI)^(-1) Φ^T Y`

5. **计算类原型** (`_compute_means`)
   - 为每个类别计算特征均值
   - 计算协方差矩阵和标准差（用于后续知识反刍）

### 增量任务 (Task 1-N) 详细流程

**目标**: 学习新类别，同时保持对旧类别的记忆

1. **网络扩展**
   ```python
   # 扩展分类头以包含新类别
   self._network.update_fc(total_classes, Hidden, cosine_fc=True)  # Cosine分支
   self._network.update_fc(total_classes, Hidden)  # AC分支
   # 冻结 AC 模型参数
   for param in self._network.ac_model.parameters():
       param.requires_grad = False
   ```

2. **适配器增量训练** (`_progreessive_train`)
   - 训练轮数: `progreesive_epoch` (默认80)
   - 创建 EMA 模型用于动量更新
   
   **损失函数组成**:
   ```python
   # 基础分类损失
   loss_ce = CrossEntropy(logits, targets)
   
   # (可选) 知识蒸馏损失 (lambda_fkd > 0)
   loss_kd = KL_Divergence(
       current_logits[:, :old_classes] / T,
       old_logits[:, :old_classes] / T
   ) * T^2
   
   # (可选) 参数正则化 (rg > 0)
   loss_reg = ||θ_new - θ_old||^2
   
   # 总损失
   loss = loss_ce + λ_fkd * loss_kd + rg * loss_reg
   ```

3. **EMA 动量更新** (每个 epoch 后)
   ```python
   # 如果 alpha > 0
   θ_ema = alpha * θ_ema + (1 - alpha) * θ_current
   # 训练结束后将 EMA 参数复制回主模型
   ```

4. **计算新类原型** (`_compute_means`)
   - 对新增的每个类别计算特征均值和统计量

5. **增量 AL 权重更新** (`IL_align`)
   - 使用递归最小二乘（RLS）算法
   - 对每个新样本批次:
     ```python
     # Sherman-Morrison-Woodbury 公式
     R_new = R_old - R_old @ Φ^T @ (I + Φ @ R_old @ Φ^T)^(-1) @ Φ @ R_old
     W_new = W_old + R_new @ Φ^T @ (Y - Φ @ W_old)
     ```

6. **知识反刍机制** (如果 `cali_weight > 0`)
   
   a. **校准原型模型** (`cali_prototye_model`)
      - 为旧类创建校准模型
   
   b. **计算新旧类关系** (`_compute_relations`)
      ```python
      # 找到每个旧类最相似的新类
      similarity = normalize(old_means) @ normalize(new_means)^T
      relations[i] = argmax(similarity[i]) + known_classes
      ```
   
   c. **构建反刍特征集** (`_build_feature_set`)
      - 收集新类的真实特征
      - 为旧类生成伪特征:
        ```python
        pseudo_feature[old_class] = 
            feature[related_new_class] - mean[related_new_class] + mean[old_class]
        ```
   
   d. **基于错误校准权重** (`cali_weight`)
      - 只对预测错误的样本进行权重更新
      - 再次应用 RLS 算法

7. **任务后处理** (`after_task`)
   - 保存当前网络为旧网络 `_old_network`
   - 用于下一任务的知识蒸馏

```python
# 提取某个类别的原型特征
def extract_class_prototype(model, class_idx, dataloader):
    model._network.eval()
    vectors = []
    with torch.no_grad():
        for _, inputs, targets in dataloader:
            if targets == class_idx:
                features = model._network(inputs.to(device))["features"]
                vectors.append(features.cpu().numpy())
    return np.mean(np.concatenate(vectors), axis=0)
```

## ⚙️ 消融实验配置

MoAL 的关键组件可以通过配置文件进行消融研究:

### 1. 无 EMA (alpha = 0)
```json
{
    "alpha": 0,
    // 其他参数保持不变
}
```
**影响**: 适配器参数不会进行动量插值，可能导致适应性下降

### 2. 无知识蒸馏 (lambda_fkd = 0)
```json
{
    "lambda_fkd": 0,
}
```
**影响**: 旧类知识保持能力下降，灾难性遗忘加剧

### 3. 无知识反刍 (cali_weight = 0)
```json
{
    "cali_weight": 0,
}
```
**影响**: 无法通过伪特征强化旧类知识，旧类性能下降

### 4. 无参数正则化 (rg = 0)
```json
{
    "rg": 0,
}
```
**影响**: 适配器参数可能过度偏移，影响特征提取稳定性

### 5. Hidden 维度影响
```json
{
    "Hidden": 10000,  // vs 15000, 20000, 25000
}
```
**影响**: 表征能力和计算开销的权衡

## ❓ 疑问点和建议

### 关键疑问

1. **预训练权重路径硬编码**
   - **问题**: `backbone/vision_transformer_adapter.py` 中权重路径需手动修改
   - **建议**: 将权重路径作为配置参数，在 JSON 文件中指定
   ```json
   {
       "pretrained_weight_path": "checkpoints/dino_vitbase16_pretrain.pth"
   }
   ```

2. **数据集路径配置不灵活**
   - **问题**: `utils/data.py` 中数据路径硬编码
   - **建议**: 支持环境变量或配置文件指定
   ```python
   data_path = os.environ.get('MOAL_DATA_PATH', args.get('data_path', '/default/path'))
   ```

3. **记忆机制实现但未使用**
   - **观察**: 配置中 `memory_size=0`，但代码包含 exemplar 构建逻辑
   - **疑问**: MoAL 是否完全不需要样本存储？还是可选功能？
   - **建议**: 文档说明是否支持 memory-based 变体

4. **多 GPU 训练稳定性**
   - **问题**: 使用 `nn.DataParallel`，可能在大规模实验中效率不高
   - **建议**: 考虑升级到 `DistributedDataParallel`

5. **评估指标**
   - **观察**: 同时输出 CNN 和 NME 准确率
   - **疑问**: 最终报告应该使用哪个指标？
   - **建议**: 文档明确说明 NME (Analytical Learning) 是主要指标

6. **知识反刍的计算开销**
   - **观察**: `_build_feature_set` 需要遍历所有新类样本
   - **疑问**: 在大规模数据集上是否有性能瓶颈？
   - **建议**: 考虑采样策略或批处理优化

7. **Ridge regression 超参数选择**
   - **观察**: `optimise_ridge_parameter` 使用固定的候选集
   - **建议**: 可以根据数据集规模动态调整搜索范围

### 使用建议

1. **首次运行建议**
   - 先在 CIFAR-100 上验证环境配置正确
   - 使用较小的 Hidden 维度（如 10000）快速测试
   - 确认日志输出和准确率符合预期

2. **超参数调优建议**
   - `alpha`: 大数据集建议 0.999，小数据集可以降低到 0.99
   - `Hidden`: 根据 GPU 显存调整（15000需要约10-12GB显存）
   - `lambda_fkd`: 任务数较多时建议增大（0.5-1.0）

3. **硬件配置建议**
   - CIFAR-100: 单卡 1080Ti/2080Ti (11GB) 即可
   - ImageNet-R/A: 建议 3090/V100 (24GB)
   - OmniBenchmark: 建议 A100 (40GB) 或多卡并行

4. **实验时间估计**
   - CIFAR-100 (10 tasks): ~8-10 小时 (单卡 3090)
   - ImageNet-R (10 tasks): ~20-24 小时 (单卡 3090)
   - 主要时间消耗在适配器微调阶段

## 🔧 常见问题排查

### 1. CUDA Out of Memory
**解决方案**:
- 减小 `batch_size` (48 → 24)
- 减小 `Hidden` 维度 (15000 → 10000)
- 使用梯度累积

### 2. 数据集加载失败
**检查项**:
- `utils/data.py` 中路径配置是否正确
- 数据集目录结构是否符合要求（train/, val/）
- CIFAR-100 需要网络连接以自动下载

### 3. 预训练权重加载失败
**检查项**:
- 权重文件是否下载完整
- `checkpoints/` 目录是否存在
- `vision_transformer_adapter.py` 中路径配置是否正确

### 4. 准确率异常低
**可能原因**:
- 预训练权重未正确加载
- 学习率设置不当
- 数据增强过强或过弱
- Ridge regression 参数选择失败

### 5. 训练过程卡住
**可能原因**:
- `num_workers` 设置过高，尝试减少到 4 或 0
- 数据加载瓶颈，检查磁盘 I/O

## 🎓 总结

MoAL 方法通过以下核心技术实现了类增量学习中的 **plasticity-stability** 平衡：

1. **Analytical Learning**: 递归最小二乘求解，高效获取新知识
2. **Momentum-based Adaptation**: EMA 机制实现平滑的权重过渡
3. **Knowledge Rumination**: 通过伪特征反刍强化旧知识

实验复现的关键在于:
- ✅ 正确配置预训练权重和数据集
